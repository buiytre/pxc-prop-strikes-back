Envíos: Control envía a Juego

SetZoneToServerSend - Se envían las zonas iniciales que cargarán los servidores en su encendido.
	Params:
	- idZone: id de la zona a cargar
	- idServer: id del server receptor, esta id se guarda para que el server sepa 'quien es', pues cuando se pida la carga de las zonas, a control le hará falta saber la id de este

GetZoneSend - Se envía a un servidor de destino la zona que tiene que recoger
	Params:
	- idZone: id de la zona a cargar
	- idServerSource: id del servidor origen que tiene la zona

RemoveZoneSend - Se envía a un servidor la orden de dejar de gestionar la zona
	Params:
	- idZone: id de la zona a eliminar

ServerLoadSend - Se envía a un servidor una petición para que envíe la carga de todas sus zonas
	Params:
	- Sin parámetros

Recepción: Control recibe de Juego

ServerLoadRcvd - Control recibe la carga de una zona
	Params:
	- idServer: id del server que pasa la zona(id enviada en SetZoneToServerSend  )
	- idZone: id de la zona cuya carga se está pasando
	- zoneLoad: carga de la zona que se está pasando
	- remainingZones: zonas restantes que quedan para pasar (recuerdo que se tiene que control tiene que recibir un datagrama por cada zona que tiene el servidor de juego,
en el datagrama en que el remainingZones devuelto sea 0, Control dejará de esperar nuevas zonas)
